# Telecom_Test: Dungeon Crawler Simulator

Консольное приложение на языке C++17 для симуляции работы автоматического агента (бота), исследующего подземелье с целью сбора ресурсов в условиях ограниченного запаса еды.

---

## 📋 Основные возможности

1. **Строгий синтаксический парсинг (`main.cpp`):** Полная валидация входного файла `in.txt` с обработкой аномалий (лишние пробелы, нечисловые символы вроде `2|5`, некорректный счетчик токенов).
2. **Валидация графа:** Проверка двунаправленности переходов между комнатами подземелья. При обнаружении любой ошибки симулятор локализует её и выводит дефектную строку.
3. **Двухфазный ИИ-агент (`RegularBot`):**
   * **Фаза исследования (EXPLORE):** Бот исследует подземелье до тех пор, пока не израсходует половину начального запаса еды, передвигаясь к наименьшим ID смежных или ближайших по графу комнат (через алгоритм BFS). Первый сбор в комнате бесплатен.
   * **Фаза возвращения (RETURN):** Бот рассчитывает кратчайший путь к старту (комната `0`) по посещенным вершинам. При наличии излишков еды бот жадно выкапывает ресурсы на своем пути, ранжируя их по ценности.
4. **Строгий учет игровых состояний (`GameManager`):** Менеджер выступает в роли независимого судьи. Он контролирует легальность ходов бота, списывает еду и корректно форматирует состояние ресурсов (`0` для начального отсутствия, `_` для собранного ресурса).
5. **Модульное тестирование:** Проект полностью покрыт базой из 24 юнит-тестов (Google Test), проверяющих навигацию, BFS, лимиты еды, обработку `std::optional` и логирование.

## 🛠️ Требования к окружению

* Компилятор с поддержкой стандарта **C++17** (GCC >= 7, Clang >= 5, MSVC 2017+).
* Система автоматизации сборки **CMake** версии `>= 3.14`.
* Подключение к сети Интернет при первой конфигурации (CMake автоматически скачает актуальный релиз Google Test через `FetchContent`).

## 🚀 Инструкция по сборке и запуску

1. **Клонируйте репозиторий и перейдите в корень проекта:**
   ```bash
   cd Telecom_Test
   ```

2. **Создайте директорию для сборки и перейдите в неё:**
   ```bash
   mkdir build
   cd build
   ```

3. **Сгенерируйте файлы сборки с помощью CMake:**
   ```bash
   cmake ..
   ```

4. **Скомпилируйте исполняемые файлы:**
   ```bash
   cmake --build .
   ```

5. **Подготовьте входной файл:**
   Поместите файл `in.txt` в папку `build` (рядом с исполняемым файлом).

6. **Запустите симуляцию:**
   * **Linux/macOS:**
     ```bash
     ./simulator_run
     ```
   * **Windows (Command Prompt / PowerShell):**
     ```cmd
     simulator_run.exe
     ```

После успешного выполнения симуляции в папке появится файл `result.txt`, содержащий подробный пошаговый трейс.

## 🧪 Запуск модульных тестов

Для запуска всех 24 автоматических тестов (проверка лабиринта, ИИ и менеджера), выполните команду в папке `build`:

```bash
ctest --output-on-failure
```
Или запустите скомпилированный бинарный файл тестов напрямую:
```bash
./tests/unit_tests
```

## 📥 Формат входных данных (`in.txt`)

* **Строка 1:** Число `N` — количество комнат в подземелье (не считая нулевую стартовую).
* **Последующие N + 1 строк:** Описание комнат в строгом формате:
  `<номер_комнаты> <список_смежных_через_запятую> <iron> <gold> <gems> <exp>`
  *Стартовая комната `0` может быть описана без ресурсов (только ID и соседи).*
* **Последняя строка:** Количество стартовой еды `M` и название целевого ресурса миссии (регистронезависимо), ценность которого удваивается.

**Пример корректного `in.txt`:**
```text
3
0 1,2
1 0,3 10 0 5 0
2 0 0 4 0 15
3 1 5 0 0 20
50 gold
```

## 📤 Формат выходных данных (`result.txt`)

* Перемещение бота: `go <номер_комнаты>`
* Сбор ресурсов: `collect <тип_ресурса>`
* Текущий статус ресурсов после действия (кроме возврата в `0` комнату): `state <номер_комнаты> <iron> <gold> <gems> <exp>` *(собранные ресурсы заменяются на `_`)*
* Финальный отчет при успешном возвращении: `result <iron> <gold> <gems> <exp> <общая_ценность>`

**Пример корректного `result.txt`:**
```text
go 1
state 1 10 0 5 0
collect gems
state 1 10 0 _ 0
go 0
result 0 0 5 0 115
```
